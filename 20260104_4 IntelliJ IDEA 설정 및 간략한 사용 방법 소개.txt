1. Java 프로젝트

빌드, 실행, 배포 : 자바 컴파일러 등록 및 변경, 빌드 도구
빌드, 실행, 배포 > 빌드 도구 > Maven > Maven 홈 경로 : C:\Program Files\apache-maven-3.9.12 로 설정

2) 프로젝트 설정
메인 메뉴 > 파일 > 프로젝트 구조... (Project Structure)

3) 클래스 생성
src 우클릭 > 새로 만들기 > 패키지 : com.example
com.example 우클릭 > 새로 만들기 > Java 클래스 > HelloWorld / 클래스

public static void main() 메서드 생성시, 앞글자 psvm만 치면 자동완성으로 만들 수 있다.
System.out.println() 메서드 생성시, sout만 치면 자동완성으로 만들 수 있다.

(JDK가 제대로 읽히지 않을 경우, 메인 메뉴 > 파일 > 캐시 무효화 > 모두 체크 후 "무효화 및 다시 시작" 클릭한다. 이후 다시 프로젝트 구조에서 SDK 설정하기 - JDK 설치 경로 지정)

종료 코드 0(으)로 완료된 프로세스

상단 왼쪽의 구조탭에서 전체 소스코드 구성도를 볼 수 있다.
왼쪽 하단의 버전관리탭에서 형상관리 시스템과 연결되었을 때, 해당하는 항목이 표시된다.
왼쪽 하단의 실행탭에서 자바 클래스를 실행한 결과를 볼 수 있다.
왼쪽 하단의 터미널 탭에서 윈도우라면 명령 프롬프트나 PowerShell을 사용할 수 있고, macOS라면 터미널이 연결되어 있다.
별도 프로그램 실행하지 않아도 인텔리제이 통해서 터미널을 사용할 수 있다.

얼티메이트 버전이라면 우측 상단에 Database탭에서 DB 연동도 가능하다.

상단에 콤보박스엔 실행하고자 하는 클래스를 지정 가능하다. 구성 편집...을 클릭하여 직접 클래스를 지정해서도 사용할 수 있다. 클래스를 1번 이상 실행하면 자동으로 그 클래스가 등록된다.
상단에 클래스 실행 단추와, 디버그 단추도 있다.

* 참고
[Java] var 키워드란? (간단예제포함) (https://lucas-owner.tistory.com/63)
java의 var 키워드는 java 10 부터 도입된 키워드 이다.
해당 키워드는, 지역 변수의 타입 추론을 위한 키워드 이다. 변수 선언시, 타입을 생략 가능하게 한다.

var 키워드의 경우 Compile 시점에 타입을 추론하게 된다. 즉 Compile 시점에 타입이 이미 결정된다.
이때 컴파일러가 타입을 추론할 수 없을 경우, Compile Error 가 발생하게 된다.
즉 이걸 반대로 생각한다면, RunTime 시점에 타입에 대한 연산이 일어나지 않는다는 말이다.
