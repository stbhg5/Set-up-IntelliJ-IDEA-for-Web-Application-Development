[Maven을 이용한 웹 프로젝트 생성하기]


1. Maven 프로젝트
1) 프로젝트 생성
메인 메뉴 > 파일 > 새로 만들기 > 프로젝트... > Maven 원형
이름 : myweb
위치 : ~\IdeaProjects
JDK : Eclipse Temurin 17.0.17
Archetype : org.apache.maven.archetypes:maven-archetype-webapp (메이븐에서 웹 애플리케이션 만들 수 있도록 기본적인 템플릿 제공)
고급설정)
그룹ID : org.example (사용하는 도메인(회사가 사용하는 고유 정보)으로 바꿔도 됨, 도메인 이름 뒤집어서 패키지명 사용 (도메인 : example.org) (ex. 도메인 : company.co.kr, 패키지 명 : kr.co.company))
아티팩트ID : myweb (프로젝트 또는 애플리케이션 이름)
버전 : 1.0

2) 프로젝트 생성 후 외부 라이브러리 확인
# < temurin-17 >
# Maven: junit:junit:3.8.1

3) 프로젝트 소스 확인
# pom.xml : 실제 메이븐 프로젝트 진행하며 지정할 다양한 설정 내용들이 있다.

4) 검색 기능
IntelliJ가 처음 프로젝트 로딩하면서 인덱싱 작업을 빠르게 수행한다. 따라서 원했던 파일 빠르게 찾게끔 지원해준다.
전체 검색 : Shift 2번
파일 위치 검색 : 왼쪽 상단의 "열려 있는 파일 선택" 버튼클릭시, 프로젝트에서 파일 위치 표시 / Alt + 1

5) 메이븐 설정
- pom.xml
...
  <groupId>org.example</groupId>
  <artifactId>myweb</artifactId>
  <packaging>war</packaging>
  <version>1.0</version>
...
  <dependencies>
    <dependency>
      <groupId>junit</groupId>
      <artifactId>junit</artifactId>
      <version>3.8.1</version>
      <scope>test</scope>
    </dependency>
  </dependencies>
...

packaging의 default 값은 jar 이지만, 웹 애플리케이션 서버에서 작동될 수 있는 형태라면 war 확장자를 가지도록 패키징 한다.
dependency는 현재 프로젝트에서 사용하고자 하는 라이브러리들을 관리. 외부 라이브러리 목록에 있는 junit도 dependency에 들어가 있었기 때문에 추가된 것이다. 자유롭게 라이브러리들을 추가할 수 있다.

6) 소스 코드
웹 프로젝트이기 때문에 Java 클래스 이용하여 Servlet이나 컨트롤러 생성 가능.
.jsp 파일도 볼 수 있다. 최근에는 프론트엔드와 백엔드를 나누어서 별도의 다른 언어로 개발하는 추세이기에 JSP가 예전만큼 사용되지 않음.
src - main - webapp - WEB-INF(웹 애플리케이션에 필요한 설정파일 들어감) 구조
web.xml 직접 수정해서 쓰는 케이스보다, 스프링이나 스프링부트, 스프링 MVC를 이용해서 설정들을 간소화하여 쓰는 경우가 더 많다.
